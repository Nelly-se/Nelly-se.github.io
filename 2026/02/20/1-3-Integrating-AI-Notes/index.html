1.3_Integrating_AI_Notes

Posted on 2026-02-20 In datawhale

为原型注入AI能力：当用户点击按钮时，前端整理输入并向外部的 AI 服务发起请求；接口返回结果后，再把结果展示到页面上。

API基础概念

虽然已经有过调用API的经验，但其实对API究竟是什么还不是很清楚，在这一节中，我首先明白了API可以简单理解为==你按对方要求的格式”发一个问题”，对方就按同样的格式”回一个结果”。==

同时，还了解了相关的核心要素：

  • API Key：通行证
  • Endpoint(接口路径)：API请求的具体路径，告诉服务器要访问哪个功能

完整URL=基础URL+Endpoint路径

接入AI API实战

各类AI的主要能力

  • 文本生成（LLM）：Deepseek
  • 图生文（VLM）: Qwen3 VL
    • 图像理解，描述，问答
    • 多图理解，对比分析
    • 文本提取（OCR能力）
  • 文生图：Seedream
    • 风格迁移
    • 图像变体
    • 分辨率提升
    • 图像编辑

**Qwen3 VL与Seedream串联使用 **先用 Qwen3 VL 分析参考图，理解画面内容；再用 Seedream 基于分析参考图的提示词内容生成新图片

接入API的方法

  • 生成API Key

  • 通过API文档找到调用示例

  • 将API Key和调用示例发送给AI IDE进行功能集成

  • ==调用Qwen3 VL时，需要额外编写一段图生文代码，而不能直接将curl示例丢给AI IDE。==

    Qwen3 VL 作为多模态视觉语言模型，其调用逻辑和纯文本的 DeepSeek、纯图像生成的 Seedream 存在本质差异，图片转文字的核心不是 “模型自带功能就可直接调用”，而是需要先完成图片的预处理 + 格式适配 + 多模态请求封装，这部分是 curl 示例无法直接覆盖的，而 DeepSeek/Seedream 的纯文本 / 纯图像生成请求无此额外环节，因此需要单独编写适配代码，而非直接丢 curl 示例。

    一句话：Qwen3 VL需要传的是图片数据，图片数据（二进制数据，如jpg/png）不能直接在curl中裸传（curl是基于HTTP命令行工具，仅能高效传递文本/字符串类型的参数），需要单独编写代码读取文件图片的二进制数据并将其转成Base编码的字符串（多模态接口的通用要求），把编码后的图片字符串封装到接口 要求的JSON参数中，与文本指令组合

零散知识

  • Siliconflow 硅基流动（SiliconFlow） 是国内知名的 ==AI 模型聚合平台==，提供多种主流大语言模型和视觉语言模型的 API 接口服务。

  • 如何找到当代更强的AI模型

    • LMArena
    • Artificial Analysis

  • SDK 是对 API 的封装。如果说 API 是 raw 接口，SDK 就是一套现成的工具箱——它把请求签名、错误处理、参数校验这些繁琐的细节都替你做好了。日常开发中，优先选择 SDK 而不是直接调 API，能省去不少麻烦。
